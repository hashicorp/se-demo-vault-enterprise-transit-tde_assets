Challenges Consequences
Security Model: With a weak security standard, every database server manages its own Data Encryption Key and Key Encryption Key. The attack surface for database servers inadvertently offers an additional exploit vector because exfiltrated data can be unencrypted.
Inefficient Workflow: Database servers follow manual processes to manage the lifecycle of Data Encryption Keys and Key Encryption Keys. Inconsistent practices to rotate encryption keys, re-encrypt data at rest, and validation of data integrity.
Inadequate Governance: There is no centralized audit trail to provide visibility to ensure key management lifecycle is compliant and reported. Teams do not follow authentication and authorization processes, leading to a weaker defensive posture.
